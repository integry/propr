# Review And Fix Commands

Use `/review` and `/fix` when you want AI review feedback first, then an implementation pass that applies the review comments you kept.

## `/review`

Post:

```text
/review
```

Or request specific models (one review comment per model):

```text
/review llm-claude-opus46 llm-codex-gpt55
```

The `llm-` prefix is optional in arguments. Without arguments, `/review` uses the PR review model configured in Settings.

You can add focus instructions on the lines below the command; they are included in the review prompt as additional review instructions:

```text
/review
Focus on security and error handling.
```

Reviews are read-only — the agent is instructed not to modify files. Each review comment contains:

- Findings grouped by severity: 🔴 Critical, 🟡 Warning, 🟢 Suggestion, ✅ Positive
- A final `Score: N/10` line (also used by [`/ultrafix`](./pr-ultrafix-commands.md) to decide whether its goal is reached)

## Edit Before Fixing

AI review comments are intentionally human-editable before `/fix` runs:

- Delete suggestions you do not want applied.
- Rewrite vague suggestions into clearer instructions.
- Leave only the comments you want implemented.

That lets you control what the next implementation pass applies.

## `/fix`

Post:

```text
/fix
```

Or narrow the scope — text after `/fix` (same line or below) becomes extra instructions:

```text
/fix
Only address the critical findings.
```

`/fix` collects the unprocessed AI review comments on the PR, applies them in one implementation pass, and then marks them processed so a later `/fix` does not re-apply them. Details:

- AI review comments are identified by a hidden marker that `/review` embeds in each comment; review comments that reported an error are excluded.
- Processed state is tracked per PR (retained for 30 days), so each review comment is applied at most once.
- User-authored comments are never collected by `/fix` — they are processed directly as natural follow-ups.

## Important Distinction

User-authored comments are processed directly when posted (see [PR Automation And Fine-Tuning](./pr-followup.md)). `/fix` is only for AI review comments generated by `/review`.
