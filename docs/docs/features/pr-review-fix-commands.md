# Review And Fix Commands

Use `/review` and `/fix` when you want AI review feedback first, then an implementation pass that applies the review comments you kept.

## `/review`

Post:

```text
/review
```

Or request specific models:

```text
/review <model-id-a> <model-id-b>
```

You can add focus instructions:

```text
/review
Focus on security and error handling.
```

ProPR posts one AI review comment per requested model. Reviews are read-only and do not change code.

## Edit Before Fixing

Before running `/fix`, you can edit or delete AI review comments:

- Delete suggestions you do not want applied.
- Rewrite vague suggestions into clearer instructions.
- Leave only the comments you want implemented.

That lets you control what the next implementation pass applies.

## `/fix`

Post:

```text
/fix
```

Or narrow the scope:

```text
/fix
Only address the critical findings.
```

ProPR collects unprocessed AI review comments generated by `/review`, applies them, and marks them processed.

## Important Distinction

User-authored comments are processed directly. `/fix` is only for AI review comments generated by `/review`.
