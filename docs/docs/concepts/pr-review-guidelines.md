# ProPR PR Follow-Up Rules

This page covers ProPR-specific behavior on pull requests. It is not a general code review guide.

## Normal Comments Start Follow-Up Work

When you want ProPR to change something, write a normal GitHub PR comment:

```text
Please simplify this branch and add a regression test for the empty state.
```

ProPR processes user-authored comments directly. You do not need a slash command for ordinary follow-up instructions.

## `/review` Posts AI Review Comments

Use `/review` when you want ProPR to ask one or more configured models for read-only review feedback:

```text
/review
```

Or target specific models:

```text
/review <model-id-a> <model-id-b>
```

Each model posts its own AI review comment.

## `/fix` Only Applies `/review` Comments

Use `/fix` only when you want ProPR to apply unprocessed AI review comments generated by `/review`:

```text
/fix
```

Direct user comments are already processed as follow-up work. `/fix` does not process those comments.

## Edit AI Review Comments First

Before running `/fix`, you can edit or delete AI review comments:

- Delete comments you do not want applied.
- Rewrite comments that need clearer instructions.
- Leave only the suggestions you want ProPR to implement.

## When The PR Needs More Work

Use the ProPR mechanism that matches your intent:

- Normal comment: direct change request.
- `/review`: ask for AI review feedback.
- `/fix`: apply AI review feedback from `/review`.
- `/use`: run one follow-up with a temporary model.
- `/switch`: change the PR model for future work.
- `/merge`: update the branch from the base branch.
- `/ultrafix`: run a review-fix loop.

For syntax and examples, see [PR Slash Commands](../features/pr-commands.md).
